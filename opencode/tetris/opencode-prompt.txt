## Gameplay Features

### Core Mechanics
- Standard 10x20 grid
- All 7 tetromino shapes (I, O, T, S, Z, J, L)
- 7-bag randomization system
- Gravity-based falling pieces
- Controls:
  - Move left/right
  - Rotate (clockwise at minimum)
  - Soft drop
  - Hard drop

### Collision & Placement
- Accurate collision detection (walls, floor, stacked blocks)
- Lock delay (optional but preferred)
- Spawn new piece correctly

### Line Clearing
- Detect and remove full rows
- Shift above rows downward
- Handle multi-line clears properly

### Scoring & Progression
- Score system:
  - Single, Double, Triple, Tetris
- Increase difficulty (speed up over time or per level)
- Display score and level

### Game States
- Start screen
- Playing state
- Pause/resume
- Game over + restart

## Audio System (CRITICAL FEATURE)

Use the **Web Audio API only** to generate ALL sounds and music procedurally.

### Architecture
- Create a dedicated `AudioManager` class/module
- Separate:
  - Sound Effects (SFX)
  - Background Music (BGM)
- Use a single shared AudioContext

---

## Sound Effects (SFX)

Implement synthesized sounds using oscillators + gain envelopes.

### Events & Sound Design
- Move → very short “tick” (high-frequency square wave)
- Rotate → soft click (triangle wave)
- Soft Drop → subtle repeating tone (quiet, not annoying)
- Hard Drop → strong “thump” (low-frequency sine burst)
- Piece Lock → soft impact
- Line Clear:
  - Single → short tone
  - Multi-line → richer chord or layered tones
- Game Over → descending tones
- Level Up → short ascending arpeggio

### Requirements
- Very low latency
- Volume control
- Avoid audio overlap clutter (manage polyphony)

---

## Procedural Background Music (BGM)

### Core Requirements
- Fully generated in real time (no audio files)
- Loop seamlessly
- Retro / arcade style (inspired by classic puzzle games)

### Music Design
- Tempo: ~100–140 BPM
- Time signature: 4/4
- Use a repeating pattern system

### Implementation Approach
Use Web Audio API to build a simple sequencer:

#### 1. Timing System
- Use a scheduler (setInterval or precise audio scheduling)
- Maintain BPM timing grid

#### 2. Instruments (all synthesized)
- Bass:
  - Sine or square wave
  - Simple repeating bassline
- Melody:
  - Square or triangle wave
  - Short looping motif
- Harmony (optional):
  - Light chords or intervals
- Percussion (optional but recommended):
  - Kick: low-frequency sine burst
  - Hi-hat: filtered noise

#### 3. Pattern System
- Define note sequences as arrays (e.g., frequencies or MIDI-like values)
- Loop patterns every few bars
- Slight variation every loop (optional)

#### 4. Adaptive Music (IMPORTANT)
- Increase tempo slightly as level increases
- Optionally increase intensity (more notes or layers)
