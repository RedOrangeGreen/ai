RSS Agent - Build Documentation

Overview

The rssagent directory contains a self-contained news headline fetcher using LLM (Large Language Model) with tool calling. It fetches 5 current international news headlines from BBC World News RSS feed and displays them using a locally-running Gemma 4 LLM.

Key Features:

  • Runs forever with 2-second delays between news fetches
  • Fetches headlines from BBC World News RSS feed via Python script
  • Uses Gemma 4 LLM for natural language processing
  • Easy to modify for different news sources or use cases

Step-by-Step Build Instructions

1. Transfer rssagent Directory to New Machine

# On original machine:
tar -czf rssagent.tar.gz rssagent/
scp rssagent.tar.gz user@newmachine:/home/user/

# On new machine:
cd /home/user
tar -xzf rssagent.tar.gz
cd rssagent

2. Install Prerequisites (Ubuntu/Debian)

sudo apt update
sudo apt install -y build-essential cmake git python3 curl

3. Clone and Build llama.cpp

# Clone llama.cpp (can be outside rssagent directory)
cd /home/user
git clone https://github.com/ggml-org/llama.cpp.git
cd llama.cpp

# Build with Release mode
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release -j$(nproc)

# This creates:
# build/bin/libllama.so
# build/bin/libllama-common.so

4. Download Gemma 4 GGUF Model

mkdir -p /home/user/models
cd /home/user/models

# Download using HuggingFace (requires huggingface-cli or manual download)
# Option A: Using wget (direct link may vary)
wget https://huggingface.co/ggml-org/gemma-4-E4B-it-GGUF/resolve/main/gemma-4-E4B-it-Q4_K_M.gguf

# Option B: Manual download from:
# https://huggingface.co/ggml-org/gemma-4-E4B-it-GGUF/tree/main

5. Build rssagent

cd /home/user/rssagent
mkdir build && cd build

# Option A: llama.cpp is in ../llama.cpp (relative to rssagent)
cmake ..

# Option B: Specify llama.cpp path
cmake -DLLAMA_LIB_PATH=/home/user/llama.cpp/build/bin ..

# Build
make -j$(nproc)

# Result: ./rssagent executable

6. Run the RSS Agent

cd /home/user/rssagent/build

# Make sure Python script is executable
chmod +x ../example/fetch_news.py

# Run (replace with actual model path)
./rssagent /home/user/models/gemma-4-E4B-it-Q4_K_M.gguf

# Press Ctrl+C to stop (runs forever)

Files to Modify for Customization

File Purpose What to Modify
example/rssagent.cpp Main application - Change news source (modify Python script call)
- Adjust delay: std::this_thread::sleep_for(std::chrono::seconds(2))
- Change prompt for different queries
- Modify infinite loop behavior
example/fetch_news.py RSS feed parser - Change RSS URL for different news source
- Modify XML parsing for different feed formats
- Add more headlines or filtering
src/model.cpp Model interaction - Modify Gemma 4 tool call regex (line ~190)
- Adjust model parameters (temperature, top_p, etc.)
- Change response parsing logic
src/agent.cpp Agent loop - Modify tool call handling
- Change how tool results are processed
- Add more sophisticated agent behaviors

Model Configuration

The default model configuration in rssagent.cpp:

config.n_ctx = 2048; // Context length
config.n_batch = 512; // Batch size
config.temp = 0.7; // Temperature (randomness)
config.top_p = 0.9; // Nucleus sampling
config.top_k = 40; // Top-k sampling

Modify these in rssagent.cpp main() function to change model behavior.

Troubleshooting

Common Issues

Issue Solution
llama.cpp libraries not found Set -DLLAMA_LIB_PATH=/path/to/llama.cpp/build/bin when running cmake
Python script fails to fetch RSS Check internet connection, verify Python 3 is installed with urllib and xml modules
Model fails to load Verify GGUF model path is correct, check model file is not corrupted
Gemma 4 tool call parsing fails Check regex in src/model.cpp, may need adjustment for different model output format
