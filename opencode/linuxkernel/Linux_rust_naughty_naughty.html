Memory Alignment Limitation in Vmalloc/KVmalloc

File: rust/kernel/alloc/allocator.rs

Lines: 154-157, 178-181

Description:

// TODO: Support alignments larger than PAGE_SIZE.
if layout.align() > bindings::PAGE_SIZE {
    pr_warn!("Vmalloc does not support alignments larger than PAGE_SIZE yet.\n");
    return Err(AllocError);
}

Impact: The Vmalloc and KVmalloc allocators fail with a warning (not an error) when alignment requirements exceed PAGE_SIZE. This can lead to subtle bugs where allocation appears to succeed but the alignment constraint is silently ignored.

Recommendation: Either properly support large alignments or fail more explicitly. Consider returning a more specific error type that indicates the limitation.

Description: In the into_unique_or_drop method:

While this code attempts to reset the reference count to 1 when converting an Arc to UniqueArc, it's passing a pointer to REFCOUNT_INIT incorrectly.
